ClickStarts - Quick Starts !

ClickStarts are pre-built stacks that people can build from - they are repositories that have a clickstart.json that defines what repo to clone, what app to run, if there is a database to create etc. They are designed as tools to get people started instantly with the whole stack running on the cloudbees cloud. You can fork them and make them your own, or build new ones from scratch. Look for projects postfixed with:

-clickstart

In the name, so you can easily spot them (they also have a clickstart.json file - which is the interesting bit). If you want to make your own - fork one that is similar - and go from there. Send us a pull request if you like. Look for the clickstart.json file in each repo, and be sure to read the readme.

RUNTIME CONTAINERS

CloudBees has plugin based runtime containers - these are often known as "clickstacks". You will use pre-made ones - modified ones - or build your own from scratch - you will find them all here. These are forkable - each one contains instructions on how to use if/develop it you want to customise it to your own needs.

Some clickstacks/plugins stand alone - some exist to augment others (for example - Newrelic exists to add features to other ones so you get newrelic monitoring !).

These typically end with 

-clickstack

so you can easily spot them - crazy huh ! The key file in each clickstack is the "setup" script - that is the core of every single one (a clickstack lifestyle involves installing the stack on the platform - and running setup when an application is installed. Setup is responsible for creating the control structure that the platform will use to start each container. Setup is run only once - on each app instance - when an application lands on a server. Typically the setup will write a .genapp/control/start script which will start the actual application.
